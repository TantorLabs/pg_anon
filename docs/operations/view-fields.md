## Overview

This mode displays how database fields match the anonymization rules.

## Prerequisites
- The `anon_funcs` schema with anonymization functions must already exist. See [init mode](init.md).
- A sensitive dictionary containing data about database fields and their anonymization rules must be prepared beforehand. See [create-dict (scan) mode](scan.md).

## Run example

```commandline
   python pg_anon.py --mode=view-fields \
                     --db-host=127.0.0.1 \
                     --db-user=postgres \
                     --db-user-password=postgres \
                     --db-name=source_db \
                     --prepared-sens-dict-file=sens_dict.py
   ```

> âš ï¸ **Note**
> 
> This mode can process only a limited number of fields when no filters are applied, for performance reasons.
> 
> This limit is controlled by the `--fields-count` option (default: 5000 fields).
> To avoid hitting this limit, increase the `--fields-count` value or use filter options: `--schema-name`, `--schema-mask`, `--table-name`, `--table-mask`.

## Options

### Common pg_anon options:

| Option                         | Description                                                                                                    |
| ------------------------------ | -------------------------------------------------------------------------------------------------------------- |
| `--config`                     | **Optional.** Path to the config file that can specify `pg_dump` and `pg_restore` utilities. (default: none)   |
| `--processes`                  | **Optional.** Number of processes used for multiprocessing operations. (default: 4)                            |
| `--db-connections-per-process` | **Optional.** Number of database connections per process for I/O operations. (default: 4)                      |
| `--verbose`                    | **Optional.** Sets the log verbosity level: `info`, `debug`, `error`. (default: info)                          |
| `--debug`                      | **Optional.** Enables debug mode (equivalent to `--verbose=debug`) and adds extra debug logs. (default: false) |

### View-fields mode options:

| Option                         | Description                                                                                                                                              |
|--------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------|
| `--prepared-sens-dict-file`    | **Required.** Path to a sensitive dictionary file (or list of files) previously generated by the [create-dict (scan) mode](scan.md) or created manually. |
| `--view-only-sensitive-fields` | **Optional.** Displays only sensitive fields. (default: all fields)                                                                                     |
| `--fields-count`               | **Optional.** Maximum number of fields to process for output (default: 5000)                                                                       |
| `--schema-name`                | **Optional.** Filter by schema name                                                                                                                      |
| `--schema-mask`                | **Optional.** Filter by schema name using a regular expression                                                                                                  |
| `--table-name`                 | **Optional.** Filter by table name                                                                                                                       |
| `--table-mask`                 | **Optional.** Filter by table name using a regular expression                                                                                                   |
| `--json`                       | **Optional.** Outputs results in JSON format instead of a table.                                                                                         |
